Flashing Ubuntu using QFIL

Note: It is recommended to flash using ADB. Please use this method only if the device crashed while flashing the image and the bootloader fails to start.

This document provides detailed instructions on how to flash the image to eMMC of I-Pi SMARC RB5N using QFIL on Windows.

Prerequisites (Windows)

Please make sure the following software is installed on the host device.

  • Qualcomm HS-USB QDLoader 9008 Driver

  • QPST (The QFIL tool comes bundled with this)

Please contact ipi.wiki@adlinktech.com for the image.

Board layout

  1. USB OTG connector
  2. Power jack
  3. UART Pins (See How to Read Console Output)
  4. F_DL button

Windows Host

Enabling download mode

Make sure the device is fully powered off and the USB OTG cable is not connected.

  • Hold the F_DL button and power on the device by plugging in the power cable while keeping the F_DL button depressed.

  • Release the F_DL button

  • Connect the USB OTG cable from your development host to targeted device.

  • The device should now be in download mode.

Flashing the image

  • Start the QFIL application

  1. Select your device, it will be detected as Qualcomm HS-USB QDLoader 9008.

  2. Select the “Flat Build” option

  3. Browse for the prog_firehose_ddr.elf file, which is inside the ufw folder inside your NON-HLOS image folder.

  4. Select all rawprogram_*.xml files inside the ufw folder, then select all patch*.xml files.

  5. Make sure the storage type should be set to “ufs”.

  6. Start the download.

  • After the download is finished, remove the USB OTG cable and power cable

  • For the first boot, first re-insert the USB OTG cable before re-inserting the power cable
