Module (RunBMC) Instruction

RunBMC is defined by BMC daughter board I/O specification by Open Compute Project (OCP)

  • Define the interface between the BMC subsystem and OPC hardware platforms

  • RunBMC interface through a 260pin SODIMM DDR4 connector that can mount into SODIMM socket

  • Rev1.4.1 was released in 2019 and the latest version is Rev1.5

Mechanical Dimension

  • Three dimensions in height: 32, 50, 70mmm

  • PCB thickness: 1.2mm typical

  • Maximum component height

    • on top side of PCB: 5.44mm
    • on bottom side of PCB: 1.6mm
  • ADLINK RunBMC module height is 32mm

Input and Output Interface Definitions

The table below is a comprehensible list of all signal pins supported on the 260-pin Edge connector of BMC-2600 module. The signal naming follows the OCP RunBMC Rev1.4.1 specification.

Most of pins are multi-function usage, depends on the firmware portion and applications. BMC-2600 module has the pre-defined feature set as Primary Function column shows in below.

Signal Descriptions

PCI Express

1x PCIex1 Gen2 lane

For KVM function by connection to COM module

USB

2x USB 2.0 interfaces : USB2A, USB2B

USB2A is capable of host and device mode (TBC)

USB2B is device mode and connect to COM module for KVM function

VGA

1x VGA display output interface

It functioned if the PCIe lane from BMC-2600 module connect to COM module

RGMII

1x RGMII interface

Can be used to connect a PHY on carrier board for dedicated remote management purpose Ethernet port.

RMII

1x RMII interface

Function as NC-SI usage, also depends on COM module’s specification

TACH & PWM

16x TACH and 8x PWM pins

Main usage is to drive System FAN.

I2C

13x I2C interfaces : I2C1 to I2C13

Some of them are used for specific tasks as below (TBC)

  • I2C3 used as IPMB function

  • I2C6 used as SMBus and connect to COM module

  • I2C8 connect to COM module and also connect to FRU EEPROM

  • I2C9 connect to thermal sensor which located on reference carrier board

Note: specification offers up to 16x I2C. these I2C interfaces can be used as I2C or GPIOs, depends on firmware porting. I2C14-I2C16 are porting as GPIOs usage as default

UART

5x UART interfaces : UART1 to UART5

UART2 is specifically used for Serial Over LAN function, also depends on COM module and carrier board design.

UART5 also called as CONSOLE. Dedicated for BMC console usage

GPIO

36x GPIO pins

Some of them are used for specific tasks. For example, emulated as BMC reset button, BIOS SPI_CS, THERMTRIP#, I2C_ALERT# and so on.

Please note these usages will be finalized after validation complete

Serial GPIO

1x SGPIO interface

Usage (TBC)

ADC (Analog-to-Digital Conversion) or GPI

Up to 16x ADC : ADC0 to ADC15

  • ADC0 to ADC7 are used as ADC and used for power rail monitoring, such as 5VSB, 3V3 and so on. It will be update later on

  • ADC8 to ADC15 are multi-function, either ADC or GPI. It is used as GPI as default (TBC)

SPI & FWSPI

3x SPI interface : SPI1, SPI2, SYSSPI

  • SPI1 connect to COM BIOS SPI interface, target for remote update feature, also depends on COM module and carrier board design (TBC)

  • SP2 can be used for the Firmware update of some active IC located on carrier board, depends on IC design (TBC)

  • SYSSPI can be functioned as I2C/I3C (TBC)

1x FWSPI

Dedicated for BMC chip usage, boot over flash. The flash located on BMC-2600 module.

LPC or ESPI

1x LPC or ESPI interface

Configure as LPC for COM Express module usage

Configure as ESPI for COM-HPC module usage

JTAG

1x JTAG interface (TBC)

PECI

1x PECI interface

Interfaces reserved only. COM module doesn’t offer PECI

Misc.

Up to 2x WDO (Watchdog reset output) or 2x GPIO.

These two pins are used as specific tasks (TBC)

2x RST (CPU_RST#, BMC_RESET#)

  • CPU_RST# connect to COM module’s PLRST#

  • BMC_RESET# connect to COM module’s PWROK

1x PWRGD (power good)

Used as Reset button of BMC
