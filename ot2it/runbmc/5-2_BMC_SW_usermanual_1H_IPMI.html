IPMI Command

This chapter is for the Baseboard Management Controller (BMC) typically supports various commands and instructions that allow remote management and monitoring of server hardware. The exact set of commands can vary based on the BMC vendor and the BMC firmware version. Here are some common types of commands and instructions you might find in a BMC:

  1. IPMI Commands: IPMI (Intelligent Platform Management Interface) is a standard protocol used for BMC communication. Common IPMI commands include:

    • Power control: Commands to power on, power off, or reset the server remotely.

    • Sensor readings: Commands to query temperature, voltage, fan speed, and other sensor data.

    • Event log: Commands to retrieve event logs for system events and status changes.

    • Firmware management: Commands to update BMC firmware or BIOS remotely.

    • System identification: Commands to control system LEDs or identify the physical server.

    • User management: Commands to manage user accounts and access privileges.

  2. Serial Over LAN (SOL) Commands: SOL allows console access to the server over the network. Common commands include:

  • Console redirection: Commands to start or stop SOL sessions.

  • Keyboard and video redirection: Commands to redirect keyboard and video input/output.

  • Terminal modes: Commands to adjust terminal modes and settings.When writing the user manual chapter on firmware and instruction sets, it’s essential to provide detailed descriptions of each command or instruction, including syntax, parameters, and examples of usage. This helps users effectively utilize the BMC’s capabilities for remote management and troubleshooting of server hardware.

IPMI Commands

Power control: Commands to power on, power off, or reset the server remotely.

Command Description
Power On ipmitool power on Powers on the system. This command sends a request to turn on the server.
Power Off ipmitool power off Powers off the system. This command sends a request to turn off the server immediately.
Power Cycle ipmitool power cycle Powers off and then powers on the system. This command is equivalent to a hardware reset.
Power Reset ipmitool power reset Performs a power reset on the system. This is similar to pressing the reset button on a physical server.
Power Status ipmitool power status Displays the current power state of the system. The response will indicate whether the system is on or off.
Power Soft ipmitool power soft Initiates a graceful shutdown of the system. This command attempts to shut down the system cleanly, similar to the operating system’s shutdown command.
Chassis Power Commands ipmitool chassis power <command>
Subcommands:
  status: Displays the current power state.
  on: Powers on the system.
  off: Powers off the system.
  cycle: Power cycles the system.
  reset: Resets the system.
  soft: Gracefully shuts down the system.		 Provides an alternative way to manage power using chassis-specific commands.


Sensor readings: Commands to query temperature, voltage, fan speed, and other sensor data.

IPMI Sensor Reading Commands:

Command Description
List All Sensors ipmitool sensor list Lists all available sensors and their current readings.
Get Sensor Reading ipmitool sensor get <sensor_name> Retrieves the current reading for a specific sensor. <sensor_name> is the name of the sensor you want to query.
Sensor Readings Summary ipmitool sensor Displays a summary of sensor readings, including temperature, voltage, fan speed, and other available sensors.
Temperature Sensors ipmitool sensor type Temperature Lists all temperature sensors and their readings.
Voltage Sensors ipmitool sensor type Voltage Lists all voltage sensors and their readings.
Fan Speed Sensors ipmitool sensor type Fan Lists all fan speed sensors and their readings.
Get Sensor Reading by ID ipmitool sensor reading <sensor_id> Retrieves the current reading for a sensor identified by its ID. <sensor_id> is the numerical identifier of the sensor.
Get Thresholds for a Sensor ipmitool sensor thresh <sensor_name> all Retrieves the threshold settings for a specific sensor. <sensor_name> is the name of the sensor.
Show Sensor Status ipmitool sdr Displays the Sensor Data Repository (SDR) status, showing all sensors and their current states.

These commands are essential for monitoring the health and performance of server hardware, allowing administrators to detect and respond to potential issues proactively.


Firmware management: Commands to update BMC firmware or BIOS remotely.

While specific commands vary by vendor and model, the general process involves:

Preparing the Firmware Update: Download the latest firmware updates from the vendor’s support site.

Creating Bootable Media: Use vendor-provided tools to create bootable media (ISO, USB, etc.) containing firmware updates.

Mounting or Uploading: Use IPMI commands or BMC web interfaces to mount or upload the bootable media to the BMC remotely.

Initiating Update: Run the firmware update process using the vendor’s management tools. This might involve rebooting the BMC or server to apply updates.

Note: For more details could refer to the chapter “5. BMC Firmware Update (rom.ima) of MECS7120_Build_Guide”


System identification: Commands to control system LEDs or identify the physical server.

IPMI System Identification Command

Command Description
Chassis Identify ipmitool chassis identify Activates the chassis identify LED to help physically locate the server.
Chassis Identify with Timeout ipmitool chassis identify <seconds> Activates the chassis identify LED for a specified duration. <seconds> is the number of seconds the LED should remain active.
Chassis Identify Off ipmitool chassis identify 0 Deactivates the chassis identify LED.
Chassis Status ipmitool chassis status Displays the status of various chassis components, including the system LEDs.
Set Front Panel LED ipmitool chassis frontpanel <state> Sets the state of the front panel LED. <state> can be on or off.
Get Front Panel LED Status ipmitool chassis frontpanel Retrieves the current state of the front panel LED.

These commands are useful for quickly identifying and managing specific servers within a data center or other environments where multiple systems are located.


User management: Commands to manage user accounts and access privileges.

Command Description
List Users ipmitool user list Lists all users and their IDs.
Add User ipmitool user set name <user_id> <username> Adds a new user with the specified user ID and username
Set User Password ipmitool user set password <user_id> Sets or changes the password for the specified user ID. It will prompt for the new password.
Set User Privilege Level ipmitool user priv <user_id> <privilege> <channel> Sets the privilege level for the specified user ID on a specified channel. <privilege> can be 1 (Callback), 2 (User), 3 (Operator), or 4 (Administrator). <channel> is usually 1 for LAN.
Enable User ipmitool user enable <user_id> Enables the specified user account.
Disable User ipmitool user disable <user_id> Disables the specified user account.
Delete User ipmitool user set name <user_id> "" Deletes the specified user by setting their username to an empty string.
Set User Access ipmitool user set access <user_id> <channel> <access> Configures the access settings for the specified user on the specified channel. <access> can be callin=on|off, link=on|off, ipmi=on|off.
List Channels ipmitool channel list Lists all available channels for user access.
Show User Summary ipmitool user summary Provides a summary of user accounts and their privileges.

These commands are essential for managing user accounts and their access privileges, ensuring proper security and access control for the IPMI-managed system.


Serial Over LAN (SOL) Commands:

Console redirection: Commands to start or stop SOL sessions.

Command Description
Starting SOL Session To start a SOL session using IPMI, you generally use the sol activate command. Here’s the typical syntax: ipmitool -H <BMC_IP> -U <username> -P <password> sol activate
- <BMC_IP>: Replace this with the IP address or hostname of your BMC.
- <username> and <password>: Replace these with your BMC login credentials.		 Example: ipmitool -H 192.168.1.100 -U admin -P password sol activate
This command initiates a SOL session with the BMC.
Stopping SOL Session To stop a SOL session, you typically use the sol deactivate command: ipmitool -H <BMC_IP> -U <username> -P <password> sol deactivate Example:
ipmitool -H 192.168.1.100 -U admin -P password sol deactivate
This command terminates the current SOL session with the BMC.

Additional Notes

  • IPMI Tool (ipmitool): Ensure that ipmitool is installed on your system. It’s a common command-line interface for managing BMCs and provides various capabilities including SOL session management.

  • Authentication: Replace <username> and <password> with your actual BMC login credentials.

  • BMC IP Address: Use the correct IP address or hostname of your BMC.

These commands should help you start and stop SOL sessions via the BMC using IPMI commands as outlined in the user guide provided by BMC for your specific hardware or software setup. Always refer to the latest BMC documentation or user guide for any updates or specific instructions tailored to your system.


Keyboard and video redirection: Commands to redirect keyboard and video input/output.

These commands can vary slightly depending on the specific BMC implementation, but generally, they include functionality for managing remote keyboard and video interfaces. Here’s a list of common commands and their explanations:


Keyboard Redirection Commands

Command Description
Start Keyboard Redirection start_kbd_redirection Initiates the redirection of keyboard input from the local console to the remote server.
Stop Keyboard Redirection stop_kbd_redirection Terminates the ongoing keyboard redirection session, returning control to the local console.
Send Key Stroke send_key_stroke <key> Sends a specific key stroke to the remote server. <key> is the key you want to send, such as Ctrl, Alt, Del, etc.
Send Key Combination send_key_combination <key1>+<key2> Sends a combination of keys to the remote server, such as Ctrl+Alt+Del.


Video Redirection Commands

Command Description
Start Video Redirection start_video_redirection Initiates the redirection of video output from the remote server to the local console.
Stop Video Redirection stop_video_redirection Ends the ongoing video redirection session.
Capture Screenshot capture_screenshot Captures a screenshot of the current video output from the remote server and saves it to a specified location.
Adjust Video Quality adjust_video_quality <quality> Adjusts the quality of the video redirection stream. <quality> can be values like low, medium, or high.
Full-Screen Mode toggle_full_screen Toggles between full-screen mode and windowed mode for video redirection.


General Redirection Commands

Command Description
Start All Redirection start_all_redirection Simultaneously starts both keyboard and video redirection sessions.
Stop All Redirection stop_all_redirection Simultaneously stops both keyboard and video redirection sessions.
Set Redirection Timeout set_redirection_timeout <time> Sets a timeout value for the redirection session. <time> is specified in seconds or minutes.

These commands are typically executed through a BMC interface, such as a web console or command-line utility. Ensure you refer to the specific BMC documentation for your hardware, as the exact syntax and available commands may vary.


Terminal modes: Commands to adjust terminal modes and settings.

Terminal Modes Commands

Command Description
Set Terminal Mode set_terminal_mode <mode> Sets the terminal to a specific mode. <mode> can be values like text, graphics, or secure.
Enable Echo enable_echo Enables echoing of characters typed by the user, which is useful for normal interaction.
Disable Echo disable_echo Disables echoing of characters typed by the user, often used for password entry or secure input.
Set Baud Rate set_baud_rate <rate> Sets the baud rate for serial communication. <rate> is the desired baud rate, such as 9600, 19200, etc
Set Parity set_parity <parity> Configures the parity bit for serial communication. <parity> can be none, even, odd, mark, or space.
Set Data Bits set_data_bits <bits> Sets the number of data bits per character for serial communication. <bits> can be values like 7 or 8.
Set Stop Bits set_stop_bits <bits> Configures the number of stop bits used in serial communication. <bits> can be 1 or 2.
Set Flow Control set_flow_control <type> Configures flow control for serial communication. <type> can be none, hardware, or software.
Set Terminal Size set_terminal_size <rows> <cols> Sets the size of the terminal window. <rows> and <cols> specify the number of rows and columns, respectively.
Reset Terminal reset_terminal Resets the terminal settings to their default values.
Set Terminal Encoding set_terminal_encoding <encoding> Sets the character encoding for the terminal. <encoding> can be values like UTF-8, ISO-8859-1, etc.
Enable/Disable Line Wrapping toggle_line_wrapping Toggles the line wrapping setting of the terminal.


1.2.4 Terminal Settings Commands

Command Description
Set Terminal Color Scheme set_color_scheme <scheme> Changes the terminal’s color scheme. <scheme> can be values like default, light, dark, etc.
Set Cursor Style set_cursor_style <style> Sets the cursor style in the terminal. <style> can be block, underline, or beam.
Enable/Disable Bell toggle_bell Toggles the audible bell on or off.
Set Terminal Title set_terminal_title <title> Sets the title of the terminal window. <title> is the desired title string.
Enable/Disable Scrollback Buffer toggle_scrollback_buffer Toggles the scrollback buffer on or off, allowing you to review previous output.
Set Scrollback Buffer Size set_scrollback_buffer_size <size> Sets the size of the scrollback buffer. <size> specifies the number of lines to keep in the buffer.
Set Terminal Font Size set_font_size <size> Changes the font size of the terminal text. <size> is the desired font size.
Enable/Disable Blinking Cursor toggle_blinking_cursor Toggles the blinking cursor feature on or off.

These commands help in configuring and customizing the terminal to suit specific needs and preferences. The exact syntax and availability of these commands might vary depending on the BMC implementation, so it’s advisable to refer to the specific documentation for your hardware.
